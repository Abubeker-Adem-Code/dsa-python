# Selection Sort Implementation
> Selection sort is a simple sorting algorithm and at the same time it is the slowest sorting algorithm. besides, it lacks that early exit so it checks every element every time.
### Time-complexity and space-complexity  
* **has a time complexity of O(n^2)** which makes it slow since the number of operation grows very fast as the number of items increase
>Assuming that we have 100 items it would take 10000 operations which is a lot
* **has Space Complexity of O(1)** 
> It’s very memory-efficient. We only have to use a tiny bit of extra memory to hold the min_index while we swap.

* **why i used** 
> i used it due to the small size of my list and i wanted to keep the code simple
